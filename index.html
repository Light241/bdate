This is angularjs datepicker (like [angular-ui datepicker][2]) but with one serious difference: bDate Datepicker **didn't** count any date on client side, it **works only with provided (json) data**.

But why?

There is many reasons. Mostly because sometimes you may want to set available ranges of dates from the backend. For example: you may set only aug and sept of 2014, and jan of 2015. There is do nothing with timezones - server say what is today and etc.

Common case

Simple form and simple datepicker

This is common case. Data provided with "demoData" object and result saves to the "resultModel" object.

Wait for remote data case

Datepicker wait for remote data

In this case datepicker waits for remote data (which will be loaded at 2 sec). Data provided with "delayedDemoData" object and result saves to the "resultDelayedModel" object.

Wait for remote data case

Datepicker wait for remote data

In this case two datepickers and one simple text input use same model. Data provided with "demoData" object and result saves to the "resultLinkedModel" object.
